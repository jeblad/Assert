Expect Vagrant

Minimal setup with expect role

Expect uses Mediawiki-Vagrant, and a complete setup can be made quite easily.

  1. Make sure you have Vagrant, etc, prepare a development directory, and move to that directory.
  2. Clone Mediawiki
`<span class="backtick"><code>bash
git clone --recursive https://gerrit.wikimedia.org/r/mediawiki/vagrant .
</code></span>`

3. Add role unless #535661 has been merged. (You need git-review to do this.)

`<span class="backtick"><code>bash
git review -d 535661
</code></span>`

4. Run setup.

`<span class="backtick"><code>bash
./setup.sh
</code></span>`

5. Enable role for Expect. This pulls inn role for Scribunto, which pulls in additional roles.

`<span class="backtick"><code>bash
vagrant roles enable expect
</code></span>`

6. Start the instance.

`<span class="backtick"><code>bash
vagrant up
</code></span>`

7. Done.

This should create a working environment with phpmd, phpcs, and phpunit. See scripts in composer.json for actual calls.

Additional setup

Luarocks and ldoc

It is necessary to install luarocks and a few libs to recreate the generated docs.

  1. Install luarocks. This will pull in several additional packages, the most important is lua5.1.
`<span class="backtick"><code>bash
sudo apt install luarocks
</code></span>`

2. Install ldoc. This will pull in several additional packages, like penlight, markdown, and luafilesystem.

`<span class="backtick"><code>bash
sudo luarocks install ldoc
</code></span>`

3. Done.

Manual verification

A few manual checks to verify proper operation.

Run composer import to load the examples.

Open the page “Module:Export-expect”, and go to the console. Posting the example code you should get the result described in the comment.

= type( p )           -- 'table'
= mw.dumpObject( p )  -- this is the actual table

Open the page “Module:Hello-world”, and go to the console. Posting the example code you should get the result described in the comment.

= p._hello('Foo') -- 'Hi there Foo!'
= p._hello('User:Foo') -- throws an error

An alternative is to inspect the page “hello-world”, given that the test pages are imported.
